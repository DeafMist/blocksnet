masterplan_tools.models.city_model module#

The aim of this module is to create one window to get any required data for other methods. All data is gathered once and then reused during calculations.

class masterplan_tools.models.city_model.CityModel(buildings: GeoDataFrame | None = None, services: dict[geopandas.geodataframe.GeoDataFrame] = Ellipsis, roads_geometry: GeoDataFrame | None = None, water_geometry: GeoDataFrame | None = None, railways_geometry: GeoDataFrame | None = None, nature_geometry_boundaries: GeoDataFrame | None = None, city_geometry: GeoDataFrame | None = None, accessibility_matrix: GeoDataFrame = Ellipsis, transport_graph: Graph | None = None, greenings: GeoDataFrame | None = None, parkings: GeoDataFrame | None = None, city_blocks: GeoDataFrame = Ellipsis)[source]#

polygon’s perimeter to area ratio. Objects with bigger ration will be dropped.

road geometry buffer in meters. So road geometries won’t be thin as a line.

WATER_WIDTH = 1#

water geometry buffer in meters. So water geometries in some cases won’t be thin as a line.

accessibility_matrix#

if the user have pre-caluclated accessibility_matrix, else the matrix will be calculated (!) Imortant note: it takes about 40GB RAM to calculate the matris on the intermodal or walk graph for the big city like Saint Petersburg
